History

History

  • Currently all content is served through the application servers
  • This takes time that could be used to process application logic
  • All images have to be saved in all relevant formats at all times
  • This is espicially true for mobile content

Goals

Goals

  • Only store one version of an image in the permanent storage
  • Be able to serve the image even to mobile devices in the needed sizes
  • Cleanup not needed versions of content
  • Manage access to content without the application backend

Use cases

  • It allows for on the fly transformation of images
  • Images can be scaled, cropped, recolored and even water marked
  • Content can be transformed to downloadable pdfs on the fly
  • Access can be managed without access to the application server

Architecture

Architecture

The system consists of two parts:

1. A s3 access tool that allows upload and download of content

  • Only accessable from the intranet to manage the content

2. A external access point that serves content transformed for the client
